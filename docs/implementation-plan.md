# 実装方針

## 現在の状態

`app/lib/main.dart` 1ファイルにすべてのコードが入っている最小構成。

**実装済み機能:**
- タスクの追加
- タスクの完了トグル（チェック・取り消し線）
- タスクの削除
- Cupertino UI

**課題:**
- アプリを再起動するとデータが消える（永続化なし）
- コードがすべて1ファイルに集中している

---

## フェーズ1: 永続化

**目的:** アプリを再起動してもデータが残るようにする。

**使用パッケージ:** `shared_preferences`

```bash
docker compose run --rm app flutter pub add shared_preferences
```

**実装内容:**
- タスクの追加・削除・完了変更のたびに `SharedPreferences` へ保存
- アプリ起動時に保存済みデータを読み込む

---

## フェーズ2: ファイル分割・構成整理

**目的:** 機能追加に備えてコードを整理する。

**ディレクトリ構成:**

```
lib/
├── main.dart               # エントリポイントのみ
├── models/
│   └── todo_item.dart      # TodoItem モデル
├── pages/
│   └── todo_list_page.dart # 一覧画面
└── widgets/
    └── todo_tile.dart      # タスク1件分のウィジェット
```

---

## フェーズ3: 編集機能

**目的:** 追加済みタスクのテキストを変更できるようにする。

**実装内容:**
- タスクをタップするとモーダルシート（`CupertinoModalPopup` or `showCupertinoModalPopup`）が開く
- テキストを編集して保存できる

---

## フェーズ4: フィルター

**目的:** 表示するタスクを絞り込めるようにする。

**実装内容:**
- `CupertinoSlidingSegmentedControl` で「すべて / 未完了 / 完了済み」を切り替え
- 選択中のフィルターに応じてリストを絞り込む

---

## フェーズ5: スワイプ削除

**目的:** ゴミ箱ボタンをなくし、iOS らしいスワイプ操作で削除できるようにする。

**実装内容:**
- `Dismissible` ウィジェットでスワイプ削除
- 削除時に `CupertinoActionSheet` で確認ダイアログを表示（任意）

---
